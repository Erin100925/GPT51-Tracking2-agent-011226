# Skill: requirement_structuring
**Description:** 將長篇或零散的專案／標案說明整理為結構化重點，包括目標、範疇、交付物與限制。
**Parameters:** 專案說明文字、語言、輸出粒度（高階／詳細）

# Skill: scope_definition
**Description:** 明確界定專案範疇（In Scope / Out of Scope），避免範圍蔓延並支援後續計畫設計。
**Parameters:** 專案說明文字、利害關係人需求、時間與預算限制

# Skill: wbs_design
**Description:** 設計工作分解結構（WBS），將專案拆解為階段、工作包與子任務。
**Parameters:** 專案目標、範疇說明、期望的分解層級

# Skill: milestone_planning
**Description:** 規劃關鍵里程碑與交付成果節點，對應專案階段與重要決策點。
**Parameters:** 專案時程長度、主要交付物清單、約束條件

# Skill: timeline_scheduling
**Description:** 依工作分解與依賴關係產生時程安排，包含開始時間、持續時間與關鍵路徑概念。
**Parameters:** 工作項目清單、依賴關係、資源限制、優先順序

# Skill: resource_estimation
**Description:** 估算各工作項目所需的角色、人力工時與主要資源。
**Parameters:** 工作項目清單、團隊角色定義、歷史類似專案參考

# Skill: stakeholder_mapping
**Description:** 辨識並分群利害關係人，標註權力／影響力與溝通需求。
**Parameters:** 專案背景、組織結構、外部單位列表

# Skill: risk_identification
**Description:** 系統性盤點專案可能風險（技術、時程、預算、法規、溝通等）。
**Parameters:** 專案說明、關鍵假設、依賴外部因素列表

# Skill: risk_quantification
**Description:** 將風險轉換成機率與衝擊分數，便於後續優先順序判斷與熱度圖繪製。
**Parameters:** 風險清單、評分尺度（1–10）、風險分類

# Skill: mitigation_planning
**Description:** 針對高風險項目設計緩解／備援策略與負責角色。
**Parameters:** 高優先風險列表、可用資源、時間與預算限制

# Skill: dependency_modeling
**Description:** 建立工作項目間的相依關係，標示阻塞性（Blocking）與資訊性（Informational）連結。
**Parameters:** 工作項目清單、時程約束、技術或法規依賴

# Skill: roadmap_storytelling
**Description:** 將專案計畫轉換成易懂的路線圖敘事，強調階段目標與價值。
**Parameters:** 專案目標、主要里程碑、目標受眾（管理層／技術／外部單位）

# Skill: regulatory_analysis
**Description:** 解析法規或標準文件，萃取義務、時程要求與合規風險。
**Parameters:** 法規文件內容、適用地區、產品或服務類型

# Skill: data_flow_modeling
**Description:** 描述系統或流程中的資料流向、來源與目的，支援依賴圖與架構設計。
**Parameters:** 系統邊界描述、資料來源與接收端、資料類型

# Skill: kpi_definition
**Description:** 為專案或平台定義可量測的 KPI／成效指標並連結到工作項目。
**Parameters:** 專案目標、可蒐集資料種類、決策需求

# Skill: visualization_design
**Description:** 規劃儀表板與視覺化布局，決定每個視圖呈現哪些資料與互動。
**Parameters:** 使用者角色、關鍵問題、可用圖表種類

# Skill: gantt_chart_design
**Description:** 將時程結構化為甘特圖所需的資料欄位與分組方式。
**Parameters:** 工作項目清單、開始時間、持續時間、階段分組

# Skill: heatmap_design
**Description:** 為風險或績效資料設計熱度圖軸線與分級邏輯。
**Parameters:** 風險或指標清單、分數範圍、顏色與分級規則

# Skill: graph_layout
**Description:** 為相依關係圖選擇合適的節點分組與佈局策略，提升可讀性。
**Parameters:** 節點類型、邊連結密度、想突顯的路徑或集群

# Skill: dashboard_copywriting
**Description:** 撰寫儀表板上的標題、說明文字與註解，讓非技術讀者也能理解。
**Parameters:** 目標讀者、專案領域、主要訊息重點

# Skill: technical_architecture
**Description:** 從高階角度設計系統或資料架構，標示關鍵模組與介面。
**Parameters:** 業務需求、既有系統、技術限制與偏好

# Skill: ai_use_case_analysis
**Description:** 盤點專案中適合使用 AI／大數據的環節並做可行性分析。
**Parameters:** 流程步驟、可取得資料、風險與成本考量

# Skill: cost_estimation
**Description:** 以粗略或中階精度估算專案成本（人力、工具、雲服務等）。
**Parameters:** 工作分解結構、資源費率、專案期間

# Skill: change_impact_analysis
**Description:** 分析需求變更或法規更新對時程、成本與風險的影響。
**Parameters:** 變更內容、原始計畫、受影響工作項目

# Skill: feasibility_assessment
**Description:** 評估方案在技術、營運與法規面向上的可行性與風險。
**Parameters:** 擬議方案描述、現有能力、外部依賴

# Skill: meeting_minutes
**Description:** 根據討論或輸出內容整理決議、待辦事項與負責人。
**Parameters:** 會議紀錄文字、參與者名單、時間與主題

# Skill: backlog_prioritization
**Description:** 將需求或工作項目整理成待辦清單並依價值與風險排序。
**Parameters:** 功能或任務列表、商業價值、風險與工作量估計

# Skill: experiment_design
**Description:** 為不確定假設設計實驗或 PoC（Proof of Concept）計畫。
**Parameters:** 待驗證假設、可用資源、成功／失敗判準

# Skill: documentation_drafting
**Description:** 撰寫結構化文件草稿，如專案計畫書、技術說明或簡報大綱。
**Parameters:** 目標文件類型、受眾、必要章節

# Skill: localization_review
**Description:** 檢視雙語（或多語）內容的一致性與適切性，特別是專業術語。
**Parameters:** 原文、譯文、領域（醫療、金融、政府等）

# Skill: quality_check
**Description:** 審視整份專案計畫的一致性、完整性與是否符合既定標準。
**Parameters:** 專案計畫 JSON、檢查清單、品質標準或框架
